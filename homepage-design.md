# Carbon Cut Homepage Design Spec

This file translates the broader brand direction in `design.md` into a concrete homepage plan.

## Homepage Job

The homepage should do four things well:

1. establish credibility
2. explain Carbon Cut clearly
3. show relevance across the three client types
4. move users toward the `/form` flow

If a section does not support one of those jobs, it should be removed, merged, or rewritten.

## Core Message

The page should make visitors feel:

- these people know what they are doing
- this is serious climate work
- this platform can fit my context
- I can take the next step without friction

## Tone

The homepage should feel:

- credible
- slightly technical
- activist
- calm
- structured

It should not feel:

- generic startup SaaS
- dashboard-heavy
- overly corporate
- playful
- orange-led

## Visual Priorities

The design should prioritize:

1. credibility
2. CTA clarity
3. proof
4. differentiated pathways

## Color Hierarchy

- Deep green is the anchor color.
- Orange is an accent only.
- Light warm backgrounds should carry most sections.
- Dark green blocks should be used selectively to create emphasis and rhythm.
- Orange should highlight key phrases, data emphasis, or CTA heat, but should never dominate the interface.

## Suggested Homepage Structure

The homepage should use a tighter structure than the current one.

Recommended order:

1. Header
2. Hero
3. What Carbon Cut Does
4. Who It’s For
5. How It Works
6. Proof / Trust
7. Final CTA
8. Footer

Optional sections:

- FAQ, only if it resolves important objections cleanly
- Pricing, only if pricing is important to first-visit conversion
- Testimonials, only if they add specific trust rather than generic praise

## Section Specs

### 1. Header

Job:

- orient the user quickly
- provide fast access to major parts of the page
- keep the CTA visible

Rules:

- fixed header
- logo must stay prominent
- CTA to `/form`
- navigation should stay minimal
- header should feel confident and restrained, not busy

Suggested nav:

- Product
- Who It’s For
- Method
- Proof

If the implemented sections differ, the nav labels should follow the final page architecture.

### 2. Hero

Job:

- establish authority immediately
- explain the platform at a high level
- create a clear next step

Hero should communicate:

- Carbon Cut helps people and organizations measure, reduce, and act on their carbon impact with structure

Hero components:

- strong headline
- short supporting paragraph
- primary CTA to `/form`
- secondary CTA to learn more or jump to methodology/product section
- visual support that feels grounded and strategic, not dashboard-decorative

Hero tone:

- serious
- clear
- slightly technical
- activist spine without slogans

Avoid:

- vague climate buzzwords
- generic product-marketing hype
- oversized dashboard mockups that dominate the whole hero

### 3. What Carbon Cut Does

Job:

- clarify the product before the page gets more detailed

This section should answer:

- what Carbon Cut is
- what it helps users do
- why it matters

Format options:

- three-step explanation
- structured capabilities grid
- concise editorial statement plus supporting cards

The section should feel understandable and methodical, not feature-dumped.

### 4. Who It’s For

Job:

- show Carbon Cut serves different user contexts without fragmenting the brand

The three user types:

- personal
- restaurant owner
- municipality

Rules:

- show them as different pathways within one system
- give each type a distinct practical use case
- keep the cards or blocks consistent so the platform feels unified

This section is important because it handles the product complexity without making the hero too broad.

### 5. How It Works

Job:

- explain the operational logic
- reinforce competence

This should make the product feel systematic.

Good directions:

- measure
- understand
- reduce
- act

Or another equivalent sequence grounded in the real product flow.

This section should lean slightly technical without turning into product jargon.

### 6. Proof / Trust

Job:

- convert credibility from a claim into evidence

Possible proof types:

- process transparency
- trusted methodology
- measurable outputs
- selected stats
- client-type outcomes
- testimonial with substance

Rules:

- proof should feel specific
- avoid fake-looking dashboard noise
- if using metrics, they should feel grounded and controlled

This is where dark green sections can work well if used with discipline.

### 7. Final CTA

Job:

- turn interest into action

This section should feel direct and practical.

It should answer:

- what should I do next?

Recommended CTA direction:

- start the form
- find the right pathway
- begin measuring impact

Avoid:

- aggressive conversion language
- vague motivational copy

### 8. Footer

Job:

- close the experience cleanly
- provide navigation and trust signals

Keep it simple and structured.

## Typography Guidance

Typography should carry a lot of the authority.

Use:

- strong confident headlines
- clean supporting copy
- slightly editorial rhythm

Avoid:

- trendy exaggerated display styling
- overly soft friendly consumer-app typography

## Imagery Guidance

Preferred imagery:

- environmental context with realism
- grounded atmospheric textures
- selective interface previews
- operational diagrams or structured visuals if needed

Avoid:

- floating generic dashboards
- bright AI-tech gradients
- stock-photo optimism
- decorative sustainability clichés

## CTA Guidance

Primary CTA:

- always tied to `/form`
- should feel like a practical next step

Secondary CTA:

- should reduce friction
- should help users understand the platform before committing

The CTA language should sound competent, not salesy.

## Content Guidance

The copy should:

- be direct
- avoid inflated claims
- sound informed
- sound useful

The copy should not:

- read like startup hype
- use too many vague sustainability buzzwords
- overcomplicate technical details

## What To Cut From The Current Homepage If Needed

These patterns should be reduced or removed if they weaken the story:

- generic testimonials without substance
- pricing if it feels placeholder or disconnected
- sections that repeat the same claim in different wording
- decorative product cards without clear purpose

## Implementation Principle

Do not try to maximize section count.

Aim for:

- fewer sections
- stronger hierarchy
- better pacing
- clearer proof
- cleaner CTA path

## One-Line Homepage Summary

The Carbon Cut homepage should feel like a credible climate action platform that guides different kinds of users toward practical next steps with calm authority.
